Redis 哨兵（Sentinel）是 Redis 的**高可用解决方案** ，用于监控 Redis 主从节点的健康状态，并在主节点故障时自动进行故障转移（Failover）。以下是详细解释：

---

### **1. 核心功能**

#### **(1) 监控（Monitoring）**

- **实时检查** ：哨兵持续监控主节点（Master）和从节点（Slave）是否正常运行。
- **心跳机制** ：通过发送 `PING` 命令检测节点响应。

#### **(2) 通知（Notification）**

- **告警** ：当节点异常时，可通过 API 或消息通知管理员（如发送邮件、调用 Webhook）。

#### **(3) 自动故障转移（Automatic Failover）**

- **主节点故障时** ：
    1. 哨兵选举一个健康的从节点晋升为新主节点。
    2. 更新其他从节点的配置，指向新主节点。
    3. 通知客户端新主节点的地址。

#### **(4) 配置提供者（Configuration Provider）**

- **客户端接入** ：客户端通过哨兵获取当前主节点的地址，无需硬编码 IP 和端口。

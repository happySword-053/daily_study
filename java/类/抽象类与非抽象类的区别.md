一个类可以选择继承抽象类，也可以选择继承普通类，甚至可以不显式继承任何类（默认继承`Object`类）。

  

抽象类的特殊之处在于：**如果一个类选择继承抽象类，那么它必须遵守特定规则**，但 “是否继承抽象类” 本身完全由开发者决定。

### 具体规则：若子类继承抽象类，需满足两种情况之一

抽象类中可能包含**抽象方法**（未实现的方法，用`abstract`修饰），当一个类（子类）继承抽象类时：

  

1. **如果子类是「普通类」（非抽象类）**：  
    必须**重写抽象类中所有的抽象方法**（否则编译报错）。  
    这是因为抽象方法没有默认实现，普通类必须提供完整实现才能被实例化。
    

    
    ```java
    // 抽象类（含抽象方法）
    abstract class Shape {
        public abstract double getArea(); // 抽象方法（无实现）
    }
    
    // 普通子类继承抽象类：必须重写所有抽象方法
    class Circle extends Shape {
        private double radius;
        
        @Override // 必须实现getArea()
        public double getArea() {
            return Math.PI * radius * radius;
        }
    }
    ```
    
      
    
2. **如果子类是「抽象类」（用`abstract`修饰）**：  
    可以**不重写抽象类中的抽象方法**（把实现的责任交给自己的子类）。
    

    
    ```java
    // 抽象子类继承抽象类：可以不实现父类的抽象方法
    abstract class Polygon extends Shape {
        // 不实现getArea()，没问题
        public abstract int getSides(); // 甚至可以新增自己的抽象方法
    }
    
    // 最终的普通子类：必须实现所有未实现的抽象方法（包括Shape和Polygon的）
    class Rectangle extends Polygon {
        private double length;
        private double width;
        
        @Override // 实现Shape的抽象方法
        public double getArea() {
            return length * width;
        }
        
        @Override // 实现Polygon的抽象方法
        public int getSides() {
            return 4;
        }
    }
    ```
### 最直观的区别：能否被实例化

- **普通类**：可以直接通过`new`关键字创建对象（实例化）。
    
    
    
    ```java
    class NormalClass { // 普通类
        public void method() {}
    }
    
    // 合法：普通类可以实例化
    NormalClass obj = new NormalClass(); 
    ```
    
      
    
- **抽象类**：**不能直接实例化**（用`new`会编译报错），必须通过它的非抽象子类实例化。
    
    
    ```java
    abstract class AbstractClass { // 抽象类（用abstract修饰）
        public abstract void abstractMethod(); // 抽象方法
    }
    
    // 错误：抽象类不能直接实例化
    AbstractClass obj = new AbstractClass(); // 编译报错
    ```
    
      
    
    （类比 C++：包含纯虚函数的类不能实例化，和 Java 抽象类的特性一致。）
    

### 二、方法组成的区别：能否包含抽象方法

- **普通类**：只能包含**有实现的普通方法**（不能包含抽象方法）。
    
    
    
    ```java
    class NormalClass {
        public void normalMethod() { 
            // 必须有方法体（实现）
        }
        
        // 错误：普通类不能有抽象方法
        public abstract void abstractMethod(); // 编译报错
    }
    ```
    
      
    
- **抽象类**：可以同时包含：
    
    1. **抽象方法**（用`abstract`修饰，没有方法体，必须被非抽象子类重写）；
    2. **普通方法**（有方法体，子类可以选择重写或直接继承）。
    

    ```java
    abstract class AbstractClass {
        // 抽象方法（无实现）
        public abstract void abstractMethod();
        
        // 普通方法（有实现）
        public void normalMethod() {
            System.out.println("抽象类中的普通方法");
        }
    }
    ```
    
      
    
    
    
      
    
    （类比 C++：抽象类可以同时有纯虚函数（类似抽象方法）和普通虚函数 / 非虚函数（类似普通方法）。）
